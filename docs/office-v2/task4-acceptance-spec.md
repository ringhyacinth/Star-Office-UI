# 办公室重构 v2 产品验收定义（Task4）

> 版本：v1.0（今晚上线版）  
> 日期：2026-03-01  
> 适用范围：`star-office-ui` 办公室重构 v2（前端展示 + 现有后端接口联动）  
> 目标：保证“像素游戏感展示”上线可用，同时明确不是“可操控真游戏”。

---

## 0. 验收原则（先定红线）

1. **核心链路优先于视觉炫技**：数据可见、状态可信、日志可读 > 特效完整。  
2. **展示优先于控制**：这是运营/监控可视化界面，不是可玩游戏。  
3. **阻塞项一票否决**：任一 P0 不通过，结论直接 NO-GO。

---

## 1) 必须保留功能（MUST KEEP）

> 以下项均为上线阻塞项（P0），必须全部通过。

| 编号 | 功能项 | 验收标准（通过条件） | 失败影响 |
|---|---|---|---|
| M-01 | 实时状态总链路（`/live`） | 页面打开后 10 秒内出现 Agent 状态与日志；后续持续刷新，无连续 3 次请求失败 | 指挥中心失真，NO-GO |
| M-02 | 像素办公场景 + Agent 自动运动 | 场景可渲染；Agent 根据状态在部门区域“自动”移动；异常/阵亡可见明显状态差异 | 核心视觉失效，NO-GO |
| M-03 | 顶部关键操作可用 | 指挥中心/Agent 档案切换可用；高质/低配切换可用；主题切换可用；时钟显示正常 | 主流程不可操作，NO-GO |
| M-04 | 日志面板可监控 | 日志分角色渲染（用户/助手/工具/异常）；筛选与“只看异常”可用；暂停滚动可用 | 观测能力缺失，NO-GO |
| M-05 | 任务看板可用（右侧 + 卷轴） | 最新任务列表可显示；卷轴可展开/收起；进行中/待办/已完成分区可见 | 任务态不可见，NO-GO |
| M-06 | 右侧仪表盘 KPI 可用 | 在线/活跃/待办/完成率可刷新；Agent 工作负载卡片可渲染 | 管理面失效，NO-GO |
| M-07 | 异常可视反馈保留 | 任务状态变化触发完成/失败提示（占位特效可接受）；错误状态有明确提示 | 异常不可感知，NO-GO |
| M-08 | 基础健康检查可通过 | `/health` 返回 200；页面非白屏；无阻塞级 JS 致命错误 | 无法稳定运行，NO-GO |

## 2) 可接受降级（ACCEPTABLE DEGRADATION）

> 下列降级允许上线，但必须“可感知、可回退、不影响核心链路”。

### A. 可接受（不阻塞上线）

1. **特效降级到低配模式**（雨雾扫描、霓虹脉冲、部分动画频率下降）。  
2. **自动触发 low-FX**（低 FPS 时自动切到低配）并可手动恢复。  
3. **街区特效简化**（仅保留基础层与状态灯，减少动态粒子）。  
4. **Ollama 区域显示“未运行/连接失败”**，但页面其余功能正常。  
5. **外链不可达**（项目中心/金融中心/情报中心）不阻塞本页面验收。

### B. 不可接受（触发 NO-GO）

1. `/live` 长时间不可用（>30 秒无有效数据）或持续错误。  
2. 日志面板不可读（空白/卡死/严重错乱）导致无法观察异常。  
3. 任务区与仪表盘都无法显示有效任务信息。  
4. 页面白屏、主线程明显卡死、连续报错影响核心交互。  
5. 出现“可操控角色玩法”且能改变业务状态（见边界章节）。

---

## 3) 边界定义：像素游戏感展示，但**非真游戏控制**

## 3.1 定位一句话

**“看起来像游戏场景，实际是状态可视化控制台。”**

## 3.2 允许的“游戏感”

- 像素风角色、场景分层、呼吸灯、雨雾扫描、完成/失败提示动效。  
- Agent 根据状态自动寻位/移动（由数据驱动，不由用户操控）。  
- 可点击/切换的 UI 控件（标签页、筛选、按钮、卷轴展开）。

## 3.3 明确禁止的“真游戏控制”

1. 键盘/手柄/摇杆控制角色移动、攻击、碰撞。  
2. 通过点击地图热区触发“得分、胜负、闯关、奖励”闭环。  
3. 用户操作直接驱动 agent 位置来改变业务状态（例如“把人拖到某区=改状态”）。  
4. 出现“可玩目标”文案与机制（如任务通关、积分排行、连击奖励等）。

## 3.4 边界验收动作（上线前必测）

- 按方向键 / WASD / 手柄输入：**角色无可控响应**。  
- 页面无“开始游戏/操控说明/胜负目标”入口。  
- 所有业务状态变化均来自接口数据刷新，而非场景内操控。

## 4) 今晚上线验收条目（执行清单）

> 建议由：Owner（执行）+ Reviewer（复核）共同完成；每项留截图或录屏证据。

## 4.1 启动前（T-15 分钟）

- [ ] 确认发布版本（commit/tag）冻结。  
- [ ] `/health` 可用（200）。  
- [ ] 回滚版本与回滚负责人已明确。  
- [ ] 值班窗口与沟通频道确认。

## 4.2 核心功能冒烟（T-10 ~ T-0）

- [ ] 打开页面 ≤ 5 秒可见主布局（非白屏）。  
- [ ] 顶部在线/活跃统计可刷新。  
- [ ] Agent 状态与日志可随刷新变化。  
- [ ] 任务列表可展示，卷轴可展开，状态分组正确。  
- [ ] 日志筛选（全部/异常）与暂停滚动生效。  
- [ ] 主题切换与高低配切换生效。  
- [ ] Agent 档案页可打开（至少页面不报错）。

## 4.3 边界与性能（T+0 ~ T+15）

- [ ] 键盘/手柄无角色可控行为。  
- [ ] 低配模式下界面仍可读、可操作。  
- [ ] 连续观察 10 分钟无 P0 报错刷屏。  
- [ ] 出现任务完成/失败时有可见反馈（可为占位特效）。

## 4.4 上线后观察（T+15 ~ T+45）

- [ ] `/live` 和 `/tasks` 轮询持续稳定。  
- [ ] 无持续性前端致命错误。  
- [ ] 若触发降级，已记录“原因 + 影响 + 回退计划”。

## 5) 今晚上线判定标准（PASS / NO-GO）

## PASS（允许上线）

同时满足以下条件：

1. 第 1 章 MUST KEEP **全部通过**；  
2. 第 3 章边界测试 **全部通过**（无真游戏控制）；  
3. 仅出现第 2 章“A 可接受降级”；  
4. 无未处置 P0，且 P1 均有明确绕过方案与责任人。

## NO-GO（禁止上线）

满足任一条即 NO-GO：

1. 任一 MUST KEEP 失败；  
2. 出现“真游戏控制”行为（键盘/手柄可操控角色并影响业务）；  
3. `/health` 异常或 `/live` 长时间不可用；  
4. 页面白屏/核心交互不可用/致命错误持续；  
5. 无可执行回滚方案或回滚责任人缺失。
