# 办公室重构 v2 文案与命名统一表（Task14）

> 范围：`frontend/index.html`（v2 主界面）
> 
> 目标：统一界面中文名、状态词、按钮词；标注必须替换项（含 `task` 编号显示问题）。
> 
> 说明：本文仅给出文案规范与替换清单，不改代码。

---

## 1) 命名统一原则（先定口径）

1. **数据位统一称呼 `Agent`**（保留产品识别），不与“弟子/特工”混用。  
2. **页面层级名与部门名分离**：页面入口叫“战情中心”，部门仍可叫“指挥中心”。  
3. **状态词用“中性业务词”**：`进行中/已完成/异常/已取消`，避免口语化歧义。  
4. **按钮文案用动作动词**：`清空当前显示`、`导出日志`，避免实现导向词。

## 5) 必须替换项（MUST）

> 以下项不改会导致“命名不统一/识别歧义/上线观感未完成”，建议作为 **v2 文案阻塞项**。

| ID | 必须替换内容 | 现状位置（文件:行） | 必须改成 | 原因 |
|---|---|---|---|---|
| M-01 | `task` 编号直接展示（如 `task7`） | `frontend/index.html:1894,2640,2792,2845,3054,3226` | 展示中文名（如“笑面虎”），不得裸露 `taskX` | 满足“task 编号改中文名展示”要求；提升可读性 |
| M-02 | 任务名回退到原始 `task_id` | `frontend/index.html:3494,3522,3540` | 优先 `task_name/name`，无名时用中文占位（如“未命名任务”） | 卷轴区会出现编号样式，破坏中文一致性 |
| M-03 | `Agent 档案` 与 `Agent 档案馆` 混用 | `frontend/index.html:1464` vs `1611` | 统一为“Agent 档案馆” | 同一入口命名不一致 |
| M-04 | `在线 Agent` 与 `活跃 Agent`口径重叠 | `frontend/index.html:1556,1560`（且 `2585` 注释已写在线=工作中） | 文案与计算口径一致（如“在线 Agent / 工作中 Agent”二选一明确定义） | 指标歧义，影响运营判断 |
| M-05 | 占位文案直接对外 | `frontend/index.html:2462` | 改为正式战报文案（去“占位特效”） | 上线版本不应暴露占位说明 |
| M-06 | 日志空态文案双版本 | `frontend/index.html:2797,2822` | 统一为“当前筛选暂无日志” | 同场景文案不一致 |
