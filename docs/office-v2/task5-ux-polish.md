# 办公室重构 v2：UI 打磨建议（Task5）

> 评估对象：`frontend/index.html`（当前办公室主视图）  
> 目标：优先做“可立即实现”的体验提升，不改后端接口，不改业务逻辑。

---

## 0. 结论先看（按优先级）

### P0（本周可落地，纯前端样式/文案/展示逻辑微调）
1. **右侧卡片文字可读性提升**：把 8~10px 主体文本统一提升到 10~12px，减少“看不清”。
2. **指标语义修正**：当前“在线 Agent”实际等于“工作中 Agent”，建议改名或拆分，避免误判。
3. **日志区可读性与操作文案优化**：时间列缩短、按钮命名更清晰、空态文案更可执行。
4. **中文术语统一**：统一“Agent/弟子/特工”“在线/活跃/工作中”等混用问题。

### P1（体验增强，仍可快速实现）
1. **右侧卡片信息层级重排**：把“任务总览/最新任务”固定在上，“本地模型/工作负载”下沉或折叠。
2. **日志区异常聚焦增强**：异常筛选状态显性提示、异常条目视觉权重再提升。
3. **像素氛围降噪**：默认降低扫描/雨雾强度，强化“专注看数据”模式。

### P2（氛围和品牌感升级）
1. **像素氛围情境化**：根据任务状态切换场景节奏（忙时更亮、空闲更柔）。
2. **文案体系品牌化**：构建一套统一“战情中心”风格文案词典并做 A/B。

---

## 1) 右侧卡片（Right Panel）打磨建议

### 现状观察
- 卡片中大量 8~10px 字号（如状态、说明、子标题），高频阅读压力大。  
- “任务总览 / 本地模型 / Agent 工作负载 / 最新任务”权重接近，缺少主次层级。  
- 多处“自动刷新/实时状态/运行态”重复，信息密度高但有效信息比例低。

### 建议（立即可做优先）

| 建议 | 说明 | 优先级 | 可立即实现 |
|---|---|---|---|
| 字号分层重标定 | 卡片标题 12~13px，正文 11px，辅助 10px，避免 8px/9px 作为常规阅读字号 | P0 | ✅ |
| 指标语义修正 | “在线 Agent”改为“工作中 Agent”，或新增“在线(非dead)”与“工作中”双指标 | P0 | ✅ |
| 卡片主次重排 | 上：任务总览 + 最新任务；下：模型状态 + Agent 网格 | P1 | ✅ |
| 子标题去噪 | “自动刷新/实时状态/运行态”统一成一种短标签，减少重复 | P1 | ✅ |
| 任务卡突出关键字段 | 任务名 + 状态 + P 级优先显示；描述降权，避免扫读被打断 | P0 | ✅ |

### 具体落点（便于开发直接改）
- `.right-card-title`、`.right-card-sub`、`.task-item .ti-desc`、`.agent-card .ac-state` 字号体系统一。  
- KPI 文案：`在线 Agent / 活跃 Agent` 与计算口径对齐。  
- `最新任务` 中优先展示：`P级 + 状态 + 进度`，描述保持一行省略即可。

---

## 2) 日志区（Log Panel）打磨建议

### 现状观察
- 日志正文 9px + 时间列较长（完整日期时间），读起来偏挤。  
- 操作按钮文案偏“实现视角”（如“清屏仅视图”），不够“用户动作视角”。  
- “只看异常”实际含异常+告警语义，文案不够准确。  
- 空态提示偏静态（“当前筛选暂无日志”），缺少下一步指引。

### 建议（立即可做优先）

| 建议 | 说明 | 优先级 | 可立即实现 |
|---|---|---|---|
| 日志可读性提升 | 正文字号提到 10~11px；时间列默认显示 `HH:mm:ss`，完整时间放 title | P0 | ✅ |
| 操作文案改为动作导向 | “清屏仅视图”→“清空当前显示”；“暂停滚动”→“暂停自动滚动” | P0 | ✅ |
| 筛选文案更准确 | “只看异常”→“仅看异常/告警” | P0 | ✅ |
| 空态可执行化 | “当前筛选暂无日志”增加引导：`可切换筛选或关闭异常过滤` | P0 | ✅ |
| 异常态强化 | 异常 section 左边框保留，同时提高标题/图标对比度 | P1 | ✅ |

### 日志区推荐文案替换
- 去重统计：`去重统计：0 / 0` → `日志去重：保留 0 / 原始 0`  
- 空态：`当前筛选暂无日志` → `当前筛选暂无日志，可切换到“全部”查看`  
- 清空后：`仅清空当前视图（源日志未删除）` → `已清空当前显示，不影响原始日志`。

---

## 3) 像素氛围（Pixel Atmosphere）打磨建议

### 现状观察
- 已有较完整的雨/雾/扫描线/霓虹呼吸层，氛围丰富。  
- 但长时间盯盘时，动态元素偏多，容易分散对日志与任务的注意力。  
- `任务完成/失败` 的特效文案仍是“占位特效”，沉浸感断层。

### 建议

| 建议 | 说明 | 优先级 | 可立即实现 |
|---|---|---|---|
| 默认降噪一档 | 扫描线/雾层/雨丝透明度整体下调 10~20%，减少视觉疲劳 | P1 | ✅ |
| 新增“专注模式”文案与入口 | 在现有高质/低配之外增加“专注显示（弱特效）”命名引导 | P1 | ✅（轻改） |
| 任务特效文案去占位 | “任务失败/中断（占位特效）”改为正式战报文案 | P0 | ✅ |
| 状态驱动氛围（进阶） | 错误增多时偏红、完成率高时偏暖，形成“状态可视化环境” | P2 | ⏳ |

### 推荐替换（任务特效文案）
- 成功：`✅ 任务完成（占位特效）` → `✅ 战报回传：任务已完成`
- 失败：`⚠ 任务失败/中断（占位特效）` → `⚠ 战报告警：任务失败或中断`

---

## 4) 中文文案优化（统一术语 + 更像产品）

### 术语统一建议
- 统一主体称呼：**推荐“Agent”**（在叙事位可用“弟子/特工”，在数据位保持 Agent）。
- 状态词统一：在线 / 工作中 / 待命 / 异常，避免“活跃/在线/工作中”无口径混用。
- 模块语气统一：数据区用中性专业语，氛围区用轻剧情语。

### 快速替换清单（建议首批上线）

| 当前文案 | 建议文案 | 备注 | 优先级 |
|---|---|---|---|
| 指挥中心 | 战情中心 | 更贴近看板语义 | P1 |
| Agent 档案 | Agent 档案馆 | 与页面标题一致 | P1 |
| 在线 Agent | 工作中 Agent（或在线 Agent） | 与统计口径一致 | P0 |
| 活跃 Agent | 当前工作中 | 与“在线”去重 | P0 |
| 清屏仅视图 | 清空当前显示 | 用户动作更直白 | P0 |
| 只看异常 | 仅看异常/告警 | 与实际逻辑一致 | P0 |
| 导出最近200条 | 导出最近 200 条日志 | 增加对象名 | P0 |
| 检测中... | 正在检测本地模型... | 语义完整 | P1 |
| 暂无任务数据 | 暂无任务数据，请稍后自动刷新 | 给出预期 | P1 |

---

## 5) 落地顺序建议（2 周内）

### Sprint A（P0，先做）
1. 右侧与日志的字号/间距/文案统一（纯样式+文案）。
2. 指标口径修正（“在线/工作中”）。
3. 日志操作按钮与空态提示替换。
4. 任务完成/失败特效文案去“占位”。

### Sprint B（P1）
1. 右侧卡片层级重排与去噪。  
2. 日志异常聚焦与筛选反馈增强。  
3. 像素特效默认降噪 + 专注模式命名。

### Sprint C（P2）
1. 状态驱动的环境氛围联动。  
2. 文案品牌化与 A/B 迭代。

---

## 6) 验收指标（建议）

- **可读性**：15 分钟连续盯盘后主观疲劳评分下降（问卷/访谈）。  
- **定位效率**：找出“当前最高优先任务”耗时下降。  
- **日志效率**：从日志发现异常的平均时间下降。  
- **文案理解一致性**：团队对“在线/工作中/异常”口径理解一致。

---

> 总结：本轮建议重点是“**先把看清楚、看明白、看得久**”做好。P0 基本都可通过 CSS + 文案 +轻逻辑修正完成，投入小、收益快，适合作为办公室 v2 的第一批 UX 打磨。
